# Mission Control: Frontend Architecture & Specification

## 1. Overview
The **Mission Control** interface is the central command hub for the `rusty-SUNDIALS` Autoresearch platform. Designed to orchestrate planetary-scale CCU biorefineries and Exascale fusion plasma simulations (ITER), the frontend provides real-time visualization of autonomous scientific discoveries, formal proofs, and media generation.

**Tech Stack:**
*   **Framework:** React (Vite)
*   **Styling:** Custom CSS with CSS Variables (Dark/Neon Cyber-Physical Theme)
*   **Icons:** Lucide-React
*   **Deployment:** Google Cloud Run via Docker (Nginx Alpine)

---

## 2. Core Architecture & Routing

The application is structured as a Single Page Application (SPA) with an persistent layout featuring a `Sidebar` for navigation and a `TopNav` for authentication/role management.

### 2.1 Navigation & Pages
*   **`/` (Dashboard/Terminal):** The primary entry point. A unified command-line style interface (`TerminalPage.jsx`) that displays real-time telemetry from the Autoresearch agent (e.g., Slurm job dispatching, PDE convergence).
*   **`/discoveries`:** A grid-based visualization matrix (`DiscoveriesPage.jsx`). It parses the incoming JSON streams to render physical and computational breakthroughs. Includes Phase I-IV Bioreactor protocols and Phase III ITER Fusion protocols (F, G, H, I).
*   **`/publications`:** The academic pipeline (`PublicationsPage.jsx`). It retrieves the formal LaTeX reports generated by the backend, rendering abstracts, equations, and offering a `.tex` source download for peer review.
*   **`/education`:** The media and outreach hub (`EducationPage.jsx`). Houses pre-generated 8K infographics, animated Manim/Blender pipeline stubs, and press releases. Features a dedicated **ITER Fusion Master Section**.

## 3. Component Hierarchy

### 3.1 `GlowPanel.jsx`
The primary layout container. It enforces the "Cyber-Physical" aesthetic by wrapping content in a highly stylized neon-bordered container.
*   **Props:** `title` (String), `className` (String), `style` (Object), `children` (ReactNode).
*   **Behavior:** Applies hover glow states and subtle background gradients.

### 3.2 `Terminal.jsx`
A simulated command-line interface component utilized on the dashboard to visualize raw output from the Python autoresearch orchestrator.

## 4. API Layer & Serverless "Demo Mode"

To ensure the Mission Control dashboard is highly available, cost-effective, and fully functional for academic peer review without requiring an expensive persistent GPU backend, the data layer utilizes a "Mock Interception" strategy.

### 4.1 `api/client.js`
The HTTP client wrapper intercepts all outward-bound API requests (`getDiscoveries`, `getReport`). If the backend is unreachable (or explicitly disabled for serverless hosting), it intercepts the `fetch` call and serves static, formally verified data.

### 4.2 `api/mockData.js`
Contains the static payloads:
*   `MOCK_RESULTS`: Includes metrics for all K-O protocols and the F-I ITER Fusion protocols (e.g., $k_L a=310$, 40ns HDC latency, 11.4 MW/m² heat fluxes).
*   `MOCK_REPORT`: The compiled academic data containing the abstract, results, and LaTeX strings for the publication generation.

## 5. Deployment Strategy (GCP Cloud Run)

The frontend is packaged for serverless edge delivery using a multi-stage Docker build.

### 5.1 Dockerfile
1.  **Build Stage:** Uses `node:18-alpine` to execute `npm run build` via Vite, generating static HTML/CSS/JS in the `/dist` directory.
2.  **Serve Stage:** Uses `nginx:alpine` to serve the static bundle.

### 5.2 Nginx Configuration (`nginx.conf`)
*   Configured to listen on the port injected by Cloud Run (via `$PORT` environment variable substitution).
*   Implements `try_files $uri $uri/ /index.html;` to ensure React Router handles deep linking without throwing 404 errors.

## 6. Security & Legal
*   **Authentication:** Basic role-based access (`useAuth`) limits specific buttons (e.g., triggering a live GPU run) to `admin` roles, preventing malicious actors from triggering expensive GCP jobs.
*   **Copyright Enforcement:** All exported data (publications, telemetry) is legally tagged with the SocrateAI Lab & Xavier Callens copyright notice.
