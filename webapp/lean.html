Formal Verification in Rusty-SUNDIALS

How we use Lean 4 and SocrateAI to mathematically prove the correctness of our Rust numerical solver, ensuring zero undefined behavior and strict adherence to the C specification.

1. Curry-Howard Isomorphism

The foundation of Lean. This principle states that propositions are types and proofs are programs. By defining the SUNDIALS mathematical contract as a Type in Lean, any program (proof) that successfully compiles against that type is mathematically guaranteed to satisfy the contract.

2. Lean 4 Tactics

Writing raw proofs is extremely tedious. Lean provides "tactics" — metaprograms that manipulate the goal state. Tactics like simp, rw (rewrite), and apply help us break down complex differential equation invariants into manageable logical steps.

3. SpecToRust Pipeline

SocrateAI translates the SUNDIALS C code into formal Lean 4 specifications. Once the properties are proven in Lean, the neuro-symbolic engine generates idiomatic Rust code that strictly adheres to the verified invariants, bridging the gap between math and hardware.
